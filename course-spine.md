# Course Spine

The semester is organized around four questions.

## 1. What Is The Thermal Condition?

Not "the room is 24 C."

Ask:

- Which location?
- Which hour?
- Which surface?
- Which body?
- Which weather sequence?
- Which model?
- Which comfort or protection target?
- Which uncertainty source?

Core lesson:

> A thermal condition is not a number. It is a sampled field.

## 2. Where Does It Vary?

Spatial heterogeneity is the architectural core.

Examples:

- indoor versus outdoor
- air-conditioned interior versus shaded threshold
- outdoor sun patch versus tree shade
- perimeter versus core
- roof-exposed versus middle-floor
- sunlit versus shaded
- glass-adjacent versus interior
- courtyard versus street canyon
- ground surface versus air temperature
- radiant asymmetry versus air-based comfort

Core lesson:

> The plan, section, surface, and body all matter.

## Portable Physics Thread

Students should learn a small set of equations well enough to use them in architectural reasoning:

| Physics idea | Equation / concept | Architectural consequence |
|---|---|---|
| spatial difference | `Delta T = T_B - T_A` | one room/threshold is not one condition |
| conduction | `q = U A Delta T` | envelope area, U-value, and temperature difference matter |
| stored heat | `Q = m c Delta T` | material capacity creates thermal build-up and lag |
| heat balance | `C dT/dt = gains - losses` | shade, ventilation, insulation, and internal gains change time behavior |
| accumulated exposure | `degree-hours = sum max(T - T_threshold, 0)` | duration and tail hours matter |
| radiant exchange | `q_rad approx h_r A (T_surface - T_body)` | surface temperature and body position change perceived heat |

## Indoor/Outdoor Continuity

The course should not split indoor comfort and outdoor microclimate into separate worlds. Students should learn that both are thermal fields shaped by radiation, air movement, surface temperature, exposure, duration, clothing, activity, and control.

Useful paired situations:

| Indoor condition | Outdoor or threshold analogue |
|---|---|
| glass-adjacent seat | sun-exposed bench |
| cold interior surface | high-emissivity night-sky exposure |
| radiant asymmetry beside glazing | one-sided solar exposure at a bus stop |
| stratified atrium | shaded courtyard with weak air movement |
| roof-exposed room | roof terrace or top-floor walkway |
| mixed-mode classroom | semi-outdoor studio/corridor threshold |

## 3. When Does It Vary?

Thermal conditions unfold in sequences.

Examples:

- hourly exposure states
- diurnal swings
- heatwave periods
- shoulder seasons
- night flushing
- warm-up and cool-down behavior
- future weather scenarios
- failure hours rather than annual averages

Core lesson:

> Thermal design is choreography, not a static compliance snapshot.

## 4. What Does Design Do About It?

Design action should be explicit.

Examples:

- shade
- expose
- shelter
- insulate
- ventilate
- relocate program
- change section
- change envelope
- change material
- localize control
- introduce thermal choice
- design for refuge
- identify when the tested action ladder is insufficient

Core lesson:

> Compliance is not protection. A passed metric can hide unresolved burden.

## Technical Idea To Architectural Object

| Technical idea | Architectural object |
|---|---|
| Mean radiant temperature | window seat, atrium edge, shaded bench, glass facade |
| Surface temperature | material choice, pavement, wall assembly, exposed slab |
| Solar radiation | massing, orientation, canopy, facade depth |
| Air movement | section, operable openings, courtyard geometry |
| Thermal heterogeneity | plan, section, threshold, program placement |
| Uncertainty | sensors, simulations, weather files, comfort surveys |
| Adaptive comfort | user control, mixed-mode operation, seasonal occupation |
| Vulnerability | housing, schools, elder care, public space, heat refuge |

## Thermal Finding To Architectural Translation

| Thermal finding | Architectural translation |
|---|---|
| Perimeter zone overheats in afternoon | external shading, lower SHGC, program relocation, facade depth |
| Air temperature passes but MRT fails | surface temperature control, glazing strategy, radiant asymmetry mitigation |
| Roof-exposed room fails during heatwave | roof insulation, cool roof, ventilated cavity, night purge, ceiling fan |
| Average comfort passes but tail hours fail | design for extreme periods, not annual average |
| Comfort model passes but physiology/protection fails | revise target, add vulnerable occupant profile, introduce refuge or stronger intervention |
| One weather file passes, future/extreme file fails | climate stress-test, robust rather than optimized design |
| Strategy grid cannot reach target | report censored result; change intervention class |
| Retrofit reduces energy but worsens tail discomfort | reject or revise strategy; add risk constraint |
| Outdoor shaded area works only at certain hours | redesign canopy, orientation, tree placement, occupancy schedule |
| Sensor evidence is ambiguous | improve protocol, add MRT/surface measurement, state uncertainty |

## Sensing And Radiant Translation

The sensing module should make radiant heterogeneity physically legible:

1. measure air temperature, globe/SMART radiant signal, air speed when possible, humidity, and surface temperatures
2. translate the sensor reading into mean radiant temperature or radiant flux logic
3. compare sensed heat against plan/section position
4. identify what the sensor sees and what it misses
5. convert the result into a design statement: shade, surface, material, distance, orientation, or use pattern

# Deliverables Scaffold

This file defines the student-facing production logic.

The course should feel like students are assembling a coherent thermal argument, not being asked to restart at the final project.

## Core Deliverable Logic

Final project:

> one thermal condition, one failure mode, one bounded design action.

Not:

- whole-building redesign
- decade-by-decade climate retrofit study
- professional HVAC report
- software portfolio
- generic comfort poster

## Accumulated Thermal Evidence Deck

Each student builds a small deck across the semester.

| Week range | Artifact | Format | What it proves |
|---|---|---|---|
| Weeks 1-3 | Condition Card | 1 page | the thermal condition is specific enough to study |
| Weeks 1-3 | Spatial Air Field Map | plan/section + Ta/RH readings | the air/humidity field varies across position |
| Weeks 4-5 | Radiant Exchange Card | plan/section + radiant/surface evidence | same air temperature can mean different body heat exchange |
| Weeks 6-9 | Thermal Build-Up Audit | sequence + threshold plot | heat accumulates, lags, or exceeds thresholds over time |
| Weeks 9-10 | Design Action Ladder | 3-step ladder or small matrix | the student has a bounded action space |
| Weeks 10-12 | Roll-Forward Validation Note | one changed stress/window assumption | the student understands how climate stress can weaken or confirm the action |
| Weeks 10-12 | Design Action Memo | 2 pages or 2 slides | evidence becomes one architectural decision |
| Week 12 | Reproducibility Capsule | folder or appendix | another person can inspect or rerun the evidence |

## What Each Artifact Should Look Like

### Condition Card

- one photo, drawing, or plan/section crop
- location and scale
- body position or use condition
- one thermal claim
- one hidden default

### Spatial Air Field Map

- plan or section with sample points
- air temperature and relative humidity readings or inferred values
- time stamp
- uncertainty note
- what the map cannot prove

### Radiant Exchange Card

- compared body positions
- surface temperature or radiant/MRT proxy
- air temperature comparison
- body orientation
- heat-flux sketch
- one architectural implication

### Thermal Build-Up Audit

- 3-5 frames
- time or exposure labels
- one time-series plot
- threshold line
- degree-hour, exceedance, or accumulated exposure note
- explanation of capacity, lag, gain, or loss
- one candidate action

### Failure Mode Card

- original claim
- challenged metric or assumption
- failure type
- evidence
- consequence for design

### Design Action Ladder

Three steps maximum for the required version:

1. use, occupancy, schedule, position, or control access
2. spatial, surface, shade, material, vegetation, or facade action
3. envelope, ventilation, fan, mixed-mode, or control action

The ladder can report:

- found: target reached within the tested action space
- censored: target not reached within the tested action space
- unresolved: evidence is insufficient to claim found or censored

### Design Action Memo

The memo says:

1. what should change
2. why this action is supported
3. which evidence matters most
4. how the action behaves under a changed stress/window assumption
5. what the action does not solve
6. what should be tested next

### Roll-Forward Validation Note

Minimum:

- one changed assumption: hotter week, future weather file, different stress window, altered occupancy, or changed design/weather boundary
- same action tested or reasoned against the changed assumption
- result label: robust, weakened, censored, or unsupported
- one future-proofing implication
- one limitation owned honestly

Distinction-level:

- two or more weather/time windows
- segment-length sensitivity, such as 3-day versus 7-day versus 14-day heat period
- different glazing/window assumptions while keeping the comparison controlled
- clear statement of whether the design conclusion survives the roll-forward test

### Reproducibility Capsule

Minimum:

- source list
- data table or CSV
- notebook/workbook/script/model file if used
- exported key figure
- short rerun or inspection instructions
- known assumptions and limits

## Meaningful Final Output Options

Students can choose the final communication type that best fits the condition.

| Output type | Best for | Typical package |
|---|---|---|
| Thermal object card | one clear spatial claim | one-page figure + action memo |
| Design decision memo | studio-facing recommendation | two slides + evidence appendix |
| Sensing atlas | fieldwork-heavy condition | field map + sequence + uncertainty |
| Intervention card set | multiple small actions | action ladder + before/after cards |
| Microclimate section | outdoor/threshold condition | section sequence + action memo |
| Simulation state dossier | model-heavy condition | hourly outputs + failure card + action memo |
| Portable physics sheet | equation-heavy condition | small equation + plot + design translation |
| Roll-forward validation card | climate/future-proofing condition | stress-window comparison + design implication |

## Workload Control

The final package should be bounded by these rules:

- one primary condition
- one primary failure mode
- one primary design action
- one required stress test
- one required roll-forward validation note
- one portable equation used correctly
- maximum three required intervention steps
- optional multi-window/weather-file sensitivity only when setup allows
- maximum two final boards plus reproducibility appendix, unless the DCP later requires another format
